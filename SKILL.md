# 中文小说写作规范 (novel-writer)

本 skill 规范 AI agent 在编写、修改、扩展中文长篇小说时的工作流程。所有规则适用于以"卷-章节"结构组织的小说项目。

## 项目目录结构（强制）

每个小说项目的根目录必须遵循以下结构：

```
小说根目录/
├── 全局大纲.md            # 必需：整本小说的总大纲，按卷分割
├── 人物设定/              # 必需：角色设定文件夹
│   ├── 主角名.md
│   ├── 配角名.md
│   └── ...
├── 第1卷卷名/             # 各卷目录，命名为"第N卷卷名"
│   ├── 卷大纲.md          # 必需：本卷大纲，按章节分割
│   ├── 第1章章节标题.md
│   ├── 第2章章节标题.md
│   └── ...
├── 第2卷卷名/
│   ├── 卷大纲.md
│   └── ...
└── ...
```

开始任何写作动作前，必须先确认上述结构存在；若缺失，先与用户讨论并创建对应文件，再继续。

## 一、维护规范：全局大纲与人物设定

### 全局大纲（`全局大纲.md`）

- 必须存在于小说根目录
- 用简洁的语言描述整部小说的故事发展和主旨
- **必须按卷分割**，每卷一个章节，标题形如 `## 第N卷 卷名`
- 每卷描述至少包含：核心冲突、主要剧情走向、本卷主旨

### 人物设定（`人物设定/`）

- 必须以"人物设定"命名的文件夹
- 每个主要角色独立一个 `.md` 文件，文件名即角色名（如 `人物设定/林夏.md`）
- 每个角色文件**必须详细描述**：
  - **性格**：核心性格特点、行为偏好、说话方式
  - **背景**：出身、家庭、关键经历
  - **外貌**：体貌特征、穿着风格、辨识度细节
  - **在小说中经历的故事**：随剧情推进持续更新

### 一致性强制：剧情/设定冲突的处理流程

**触发条件**：用户对话中提到的剧情、角色行为或设定，与现有的全局大纲或人物设定相悖。

**处理流程（必须严格执行）**：

1. 检测到冲突时，**立即停止当前写作动作**
2. 用一段话明确指出冲突点：
   - 用户提到的内容
   - 现有大纲/设定中的描述
   - 具体的矛盾点在哪里
3. 询问用户：是否修改大纲/人物设定以适配新剧情？
4. 等待用户回复后处置：
   - 用户**同意修改** → 修改对应的大纲或人物设定文件，再继续写作
   - 用户**拒绝修改** → 调整新剧情以符合现有设定，或与用户进一步讨论
5. 修改完成后，确认本次写作未破坏其他文件的一致性

**严禁**：在未询问用户的情况下，自行选择"采纳新剧情"或"忽略新剧情"。

## 二、卷编写规范

### 新增卷的工作流（强制顺序）

**严禁**：在没有讨论卷大纲的前提下，直接创建卷目录或开始写章节。

**必须遵循的步骤**：

1. 与用户讨论本卷的核心冲突、主要剧情、章节划分（建议给出章节数量与每章主旨）
2. 用户确认大纲后，创建卷目录：`第N卷卷名/`（数字使用阿拉伯数字，如 `第3卷北境风云`）
3. 在卷目录下创建 `卷大纲.md`，写入与用户讨论一致的大纲内容
4. **同步更新** `全局大纲.md`，新增本卷概述部分
5. 完成上述四步后，才能开始写本卷任意章节正文

### 卷大纲（`卷大纲.md`）

- 必须存在于每个卷目录下
- **必须按章节分割**，每章一个二级标题，形如 `## 第N章 章节标题`
- 每章用简洁的语言描述：
  - **核心剧情**：本章发生的关键事件
  - **主旨**：本章承担的主题作用（推动主线、塑造人物、铺垫伏笔等）
  - **节奏**：开端、转折、结尾的安排

## 三、章节编写规范

### 章节正文与卷大纲的强制对应

**核心原则**：章节正文的剧情和节奏必须与卷大纲中对应章节的描述**完全保持一致**。

执行流程：

1. **写作前**：读取 `卷大纲.md` 中对应章节的描述
2. **写作中**：严格按照卷大纲的剧情走向、节奏推进；细节、对话、描写可自由发挥，但骨架不可偏离
3. **写作后**：核对正文与卷大纲是否对应；不对应则按"反向同步规则"处理

### 用户修改正文的反向同步规则

**触发条件**：用户对已写好的章节正文做出实质性修改（剧情走向、关键事件、人物行为、节奏等）。

**必须执行的反向同步**：

1. 识别用户修改的具体内容
2. **立即修改 `卷大纲.md`** 中对应章节的描述，使其与新正文一一对应
3. 如修改影响后续章节的剧情，主动询问用户是否需调整后续章节的卷大纲
4. 如修改影响人物（性格、关系、经历），同步修改 `人物设定/` 中的对应文件
5. 如修改影响整卷或全书走向，同步修改 `全局大纲.md`

**严禁**：保留与正文不一致的旧大纲，让大纲与正文脱节。

### 字数检查（每章必做）

**触发条件**：完成一章正文的编写或大幅度修改后。

**必须执行的步骤**：

1. 使用 `wc -m` 统计字符数：

   ```bash
   wc -m "第1卷某卷名/第3章章节标题.md"
   ```

2. 检查字符数是否在 **2500-4500** 之间
3. 处理结果：
   - **少于 2500**：扩写。补充细节描写、对话、心理活动、环境描写等，但不得偏离卷大纲
   - **多于 4500**：精简或拆分。先尝试删减冗余描写；若内容确实承载过多剧情，与用户讨论是否拆为两章（拆章必须同步更新 `卷大纲.md`）
   - **2500-4500 之间**：通过

说明：`wc -m` 统计字符数（含标点、换行）。中文小说习惯将其视为"字数"。如需排除空白字符以更精确，可用：

```bash
tr -d '[:space:]' < "第1卷某卷名/第3章章节标题.md" | wc -m
```

## 四、写作前/写作后检查清单

每次开始写作或修改章节时，按顺序检查：

```
写作前：
- [ ] 已读 全局大纲.md，理解全书走向
- [ ] 已读 本卷 卷大纲.md，理解本卷剧情
- [ ] 已读 涉及角色的 人物设定/角色名.md
- [ ] 用户提到的内容与上述文件无冲突；如有冲突，已询问用户

写作中：
- [ ] 严格遵循卷大纲对该章的剧情和节奏描述

写作后：
- [ ] 已用 wc -m 统计字数，结果在 2500-4500 之间
- [ ] 用户修改正文后，已同步修改 卷大纲.md（必要时连带人物设定、全局大纲）
- [ ] 全局大纲、卷大纲、人物设定、章节正文四者无矛盾
```

## 五、关键禁止事项

- **禁止**在没有 `全局大纲.md` 的情况下开始写小说；应先与用户讨论并创建
- **禁止**在没有 `卷大纲.md` 的情况下写章节；应先与用户讨论并创建
- **禁止**自行决定与已有大纲/设定相悖的新剧情；必须询问用户
- **禁止**保留大纲与正文不一致的状态（任一文件被修改后，必须双向同步）
- **禁止**跳过 `wc -m` 字数检查
- **禁止**在新增卷时跳过"先讨论卷大纲再建目录"的顺序
